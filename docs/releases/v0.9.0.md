# v0.9.0 Release Recap

v0.9.0 is a non-breaking release that adds a knowledge acquisition planner. It turns gap detection and weak-claim signals into prioritized tasks with prompts for research, crawl, verification, or refresh work.

## What Changed

| Area | New in v0.9.0 | Why it matters |
|:-----|:--------------|:---------------|
| Go SDK planner | `NamespaceHandle.AcquisitionPlan(ctx, req)` | Lets applications ask what information should be acquired next |
| REST planner | `POST /v1/namespaces/{ns}/acquisition/plan` | Gives operators a simple task plan for filling gaps and verifying weak claims |
| GraphQL planner | `acquisitionPlan(namespace:, budget:)` | Gives debugger UIs and agents a graph-friendly acquisition backlog |
| Feature metadata | `knowledge-acquisition-planner` feature tag | Lets live deployments advertise planner support through introspection |

## Non-Breaking Behavior

The planner is additive. It reads existing nodes, gap reports, and active-learning signals, then returns suggested tasks without mutating stored knowledge.
