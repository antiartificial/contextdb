# v0.73.0 Release Recap

v0.73.0 is a non-breaking documentation and metadata release that adds a compact retry fatigue preset reference for dashboard and handoff builders.

## What's New

| Area | New in v0.73.0 | Why it matters |
|:-----|:---------------|:---------------|
| Retry fatigue docs | Preset reference table | Lists each built-in preset with expanded filters and intended handoff audience |
| Dashboard guidance | Stable preset names | Helps dashboards use `preset` while still showing operators what lane they are viewing |
| API docs | REST and GraphQL links to the reference | Keeps API users near the operator-facing preset table |
| Feature metadata | `review-handoff-retry-fatigue-preset-discovery-docs` | Lets live deployments advertise that preset discovery docs are part of the supported docs surface |

## Preset Reference

| Preset | Expanded filters | Intended handoff audience |
|:-------|:-----------------|:--------------------------|
| `review-overdue` | `escalation_level=review_overdue` | Review owners or coordinators handling overdue assigned or snoozed work |
| `source-trust-anomaly` | `escalation_level=source_trust_anomaly` | Source-quality reviewers investigating credibility drops and trust anomalies |
| `unassigned-review-overdue` | `owner=unassigned`, `escalation_level=review_overdue` | Queue triage or on-call reviewers claiming overdue work with no explicit owner |

## Compatibility

This release is intentionally non-breaking. It updates docs, release metadata, and package versions without changing API schemas, SDK behavior, storage formats, or retry fatigue filtering semantics.
