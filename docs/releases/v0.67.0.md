# v0.67.0 Release Recap

v0.67.0 is a non-breaking documentation release that adds operator recipes for graph-derived KV cache refresh values.

## What's New

| Area | New in v0.67.0 | Why it matters |
|:-----|:---------------|:---------------|
| Cookbook page | `docs/deployment/kv-derivation-recipes.md` | Gives operators stable naming, scope, review, and execute guidance for derived KV hot keys |
| Recent-node recipes | `contextdb repair kv-cache --derive recent-nodes` | Shows safe dry-run-first examples for session context values |
| Review checklist | Field-by-field review table | Keeps derived cache promotion explicit before `--execute` writes |
| Feature metadata | `kv-cache-derivation-recipes` | Lets live docs advertise the derivation cookbook surface |

## Compatibility

This release adds docs and feature metadata only. Existing KV refresh commands, derived recent-node payloads, doctor checks, TTL behavior, dry-run reports, and `--execute` promotion are unchanged.

## Operator Flow

```bash
contextdb repair kv-cache \
  --key context:prod:support:recent-nodes \
  --derive recent-nodes \
  --derive-namespace support \
  --derive-label SessionContext \
  --derive-limit 5 \
  --report
```

Review the derived value, then repeat the command with `--execute` when the hot-key refresh is approved.
