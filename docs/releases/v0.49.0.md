# v0.49.0 Release Recap

v0.49.0 is a non-breaking operations release that adds an opt-in local store consistency check to `contextdb doctor`.

## What Changed

| Area | New in v0.49.0 | Why it matters |
|:-----|:---------------|:---------------|
| Doctor store check | `contextdb doctor --store-consistency` | Samples local graph nodes and verifies index visibility |
| Namespace control | `--store-namespace` and `--store-sample` | Lets operators scope the consistency probe |
| Rebuild candidates | `store_consistency` report detail | Flags vector-bearing graph nodes that are not returned by vector search |
| Feature metadata | `doctor-store-consistency` | Lets live deployments advertise local store consistency checks |

## Compatibility

This release adds a read-only, opt-in local doctor check. Existing REST doctor checks, sample writes, backup marker checks, server behavior, and retrieval behavior are unchanged.

## Operator Flow

```bash
CONTEXTDB_DATA_DIR=/var/lib/contextdb \
  contextdb doctor --store-consistency --store-namespace my-app --store-sample 100
```

The check verifies fingerprint lookup for fingerprinted sampled nodes and reports vector rebuild candidates for vector-bearing nodes that are not visible through vector search.
