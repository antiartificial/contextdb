# v0.80.0 Release Recap

v0.80.0 is a non-breaking ranking release that adds integrity verification for ranking baseline artifact manifests.

## What's New

| Area | New in v0.80.0 | Why it matters |
|:-----|:---------------|:---------------|
| Manifest verifier | `contextdb eval ranking baseline manifest verify --manifest PATH --report` | Lets CI or release reviewers re-check retained ranking artifacts after manifest generation |
| Artifact integrity | Path, byte-size, and SHA-256 validation | Detects missing, edited, or replaced JSON and Markdown baseline artifacts |
| Drift report | Per-artifact verification details | Shows which version and artifact kind failed integrity checks |
| Feature metadata | `ranking-baseline-artifact-manifest-verify` | Lets live deployments advertise ranking baseline manifest verification support |

## Operator Flow

Generate the manifest from retained ranking baselines:

```bash
contextdb eval ranking \
  --baseline-retention-dir .contextdb/ranking-baselines \
  --baseline-retention-keep 5 \
  --baseline-manifest-out ranking-baseline-manifest.json
```

Verify it later:

```bash
contextdb eval ranking baseline manifest verify \
  --manifest ranking-baseline-manifest.json \
  --report
```

The verifier exits non-zero when an artifact path is unexpectedly missing, points to a directory, has different bytes, or no longer matches the recorded SHA-256 hash.

## Compatibility

This release is intentionally non-breaking. Existing ranking eval snapshots, retention reports, delete scripts, and manifest generation are unchanged.
