# v0.85.0 Release Recap

v0.85.0 is a non-breaking ranking release that adds CI annotation output for ranking baseline manifest verification failures.

## What's New

| Area | New in v0.85.0 | Why it matters |
|:-----|:---------------|:---------------|
| Failure annotations | `contextdb eval ranking baseline manifest verify --annotations` | Prints CI-friendly error lines for failed artifact checks |
| Annotation artifact | `--annotations-out PATH` | Saves failure annotations beside JSON and Markdown verification reports |
| Artifact context | File, version, kind, and validation message | Makes hash drift, missing files, and directory mistakes easier to find in CI logs |
| Feature metadata | `ranking-baseline-manifest-failure-annotations` | Lets live deployments advertise manifest failure annotation support |

## Operator Flow

```bash
contextdb eval ranking baseline manifest verify \
  --manifest ranking-baseline-manifest.json \
  --report \
  --markdown-out ranking-baseline-manifest-verification.md \
  --annotations-out ranking-baseline-manifest-annotations.txt
```

The annotation output is empty when verification passes. When verification fails, each artifact-level failure is emitted as a `::error` line suitable for GitHub Actions logs and similar CI processors.

## Compatibility

This release is intentionally non-breaking. Existing verification reports, Markdown summaries, default `ok` output, and exit behavior are unchanged unless `--annotations` or `--annotations-out` is supplied.
