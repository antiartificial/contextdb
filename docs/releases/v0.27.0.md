# v0.27.0 Release Recap

v0.27.0 is a non-breaking restore audit release. Promotion receipts can now be verified against their original artifact manifests.

## What's New

| Area | New in v0.27.0 | Why it matters |
|:-----|:---------------|:---------------|
| Receipt verifier | `contextdb snapshot receipt verify --promotion-report receipt.json --manifest backup.manifest.json` | Confirms a promotion receipt lines up with the exported artifact metadata |
| Record-count comparison | Imported line, node, edge, and source counts are compared with manifest counts | Catches mismatched or stale promotion receipts |
| Namespace consistency | Receipt namespace must match the embedded import report namespace | Helps detect edited or malformed receipts |
| Backup identity check | Receipt backup filename is compared with the manifest backup file | Links promotion receipts back to the intended artifact |
| Feature metadata | `promotion-receipt-verify` | Lets deployments advertise support through introspection |

## Compatibility

Receipt verification is additive. Existing export, verify, rehearse, import, receipt, marker, and doctor workflows keep their behavior unchanged.
