# v0.13.0 Release Recap

v0.13.0 is a non-breaking epistemics workflow release that turns source trust changes into reviewable tasks. The review queue can now emit source-trust anomaly items when a source crosses configured credibility thresholds, drops sharply, or accumulates repeated refutations.

## What Changed

| Area | New in v0.13.0 | Why it matters |
|:-----|:---------------|:---------------|
| Source anomaly tasks | `ReviewItem{Type: "source_trust_anomaly"}` | Makes suspicious source behavior actionable in the same queue as claim review |
| Go SDK thresholds | `SourceTrustThreshold`, `SourceTrustDropThreshold`, and `SourceRefutationThreshold` | Lets applications tune anomaly sensitivity per review run |
| REST thresholds | `source_trust_threshold`, `source_trust_drop_threshold`, and `source_refutation_threshold` query params | Gives operators threshold-based source review from HTTP |
| GraphQL thresholds | `reviewQueue(... sourceTrustDropThreshold:, sourceRefutationThreshold:)` | Gives debugger UIs source trust anomaly controls |
| Feature metadata | `source-trust-anomaly-alerts` feature tag | Lets live deployments advertise source anomaly review support |

## Non-Breaking Behavior

All request fields and response items are additive. Existing review queue clients continue to work. Source anomaly tasks only appear when the caller configures a relevant source threshold, drop threshold, or repeated-refutation threshold.
