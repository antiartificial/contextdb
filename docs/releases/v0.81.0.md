# v0.81.0 Release Recap

v0.81.0 is a non-breaking test and documentation alignment release for retry fatigue preset metadata.

## What's New

| Area | New in v0.81.0 | Why it matters |
|:-----|:---------------|:---------------|
| Docs drift test | `TestRetryFatigueCookbookPresetReferenceMatchesSDK` | Fails when the cookbook preset table drifts from SDK metadata |
| Preset coverage | Name, expanded filters, REST query, and GraphQL args | Keeps operator docs aligned with copyable preset examples exposed by clients |
| Feature metadata | `review-handoff-retry-fatigue-preset-docs-test` | Lets live deployments advertise the additional docs alignment coverage |

## Operator Impact

There is no runtime behavior change. The release hardens the release process so cookbook rows for `review-overdue`, `source-trust-anomaly`, and `unassigned-review-overdue` stay synchronized with `ReviewHandoffRetryFatiguePresets()`.

## Compatibility

This release is intentionally non-breaking. Existing REST, GraphQL, Go SDK, and retry fatigue preset behavior are unchanged.

## Verification

Release verification included:

```bash
go test -count=1 ./pkg/client
go test -count=1 ./...
npm run docs:build
git diff --check
```
