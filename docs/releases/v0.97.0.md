# v0.97.0 Release Recap

v0.97.0 is a non-breaking docs contract release that publishes the retry fatigue preset schema for dashboard and automation consumers.

## What's New

| Area | New in v0.97.0 | Why it matters |
|:-----|:---------------|:---------------|
| Public schema artifact | `/schemas/retry-fatigue-presets.schema.json` | Gives docs consumers a stable JSON Schema URL for retry fatigue preset metadata |
| Drift test | `TestPublicRetryFatiguePresetSchemaMatchesFixture` | Keeps the published schema byte-for-byte aligned with the embedded SDK/REST test fixture |
| API docs link | REST and cookbook pages link the schema URL | Makes the contract discoverable from the endpoint and operator recipe docs |
| Feature metadata | `review-handoff-retry-fatigue-preset-schema-publication` | Lets live deployments advertise published schema availability |

## Operator Flow

Fetch the published schema from the docs site:

```bash
curl https://antiartificial.github.io/contextdb/schemas/retry-fatigue-presets.schema.json
```

Use it to validate the `presets` array returned by:

```bash
curl "http://localhost:7701/v1/namespaces/my-app/review/handoff-webhooks/retry-fatigue"
```

## Compatibility

This release is intentionally non-breaking. Runtime behavior is unchanged; the release publishes an existing schema fixture and adds drift coverage for the docs artifact.
