# v0.87.0 Release Recap

v0.87.0 is a non-breaking ranking release that adds stable bundle output for ranking baseline manifest verification.

## What's New

| Area | New in v0.87.0 | Why it matters |
|:-----|:---------------|:---------------|
| Verification bundle | `contextdb eval ranking baseline manifest verify --bundle-dir DIR` | Writes all release-review artifacts with one flag |
| JSON report | `ranking-baseline-manifest-verification.json` | Keeps machine-readable verification evidence |
| Markdown recap | `ranking-baseline-manifest-verification.md` | Keeps a human-readable release summary |
| CI annotations | `ranking-baseline-manifest-annotations.txt` | Preserves inline failure callouts for CI logs |
| Feature metadata | `ranking-baseline-verification-bundle` | Lets live deployments advertise bundle output support |

## Operator Flow

```bash
contextdb eval ranking baseline manifest verify \
  --manifest ranking-baseline-manifest.json \
  --bundle-dir ranking-baseline-verification
```

The bundle directory is created when needed. Verification still exits non-zero when artifacts are missing unexpectedly, are directories, have byte-size drift, or fail SHA-256 checks.

## Compatibility

This release is intentionally non-breaking. Existing `--report`, `--markdown`, `--markdown-out`, `--annotations`, and `--annotations-out` behavior remains available.
