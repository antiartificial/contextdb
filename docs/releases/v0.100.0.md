# v0.100.0 Release Recap

v0.100.0 is a non-breaking docs contract release that publishes a schema catalog for dashboard and automation consumers.

## What's New

| Area | New in v0.100.0 | Why it matters |
|:-----|:----------------|:---------------|
| Schema catalog | `/schemas/index.json` | Gives integrations one stable file for discovering published JSON contracts |
| Published schemas page | `/api/schemas` | Documents the catalog and current schema inventory for operators |
| Retry fatigue links | REST API and retry fatigue cookbook links | Makes the preset schema discoverable from endpoint docs and recipes |
| Feature metadata | `published-schema-catalog` | Lets live deployments advertise schema catalog availability |
| Drift coverage | `TestPublicSchemaIndexCatalogsRetryFatiguePresetSchema` | Keeps the catalog pointed at the published retry fatigue preset schema |

## Operator Flow

Fetch the catalog from the docs site:

```bash
curl https://antiartificial.github.io/contextdb/schemas/index.json
```

The first catalog entry points to the retry fatigue preset schema at `/schemas/retry-fatigue-presets.schema.json` and records its owner, feature, status, and release provenance.

## Compatibility

This release is intentionally non-breaking. Runtime behavior is unchanged; the release adds discoverability around already-published docs contracts.

## Verification

Release verification included:

```bash
go test -count=1 ./testdata
go test -count=1 ./...
npm run docs:build
git diff --check
```

The next reliability slices should focus on a generated closure-bundle manifest, KV receipt verification fixtures, or CI annotations for schema catalog drift.
