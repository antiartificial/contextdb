# v0.29.0 Release Recap

v0.29.0 is a non-breaking operations release that adds lifecycle summary verification for backup artifact bundles.

## What's New

| Area | New in v0.29.0 | Why it matters |
|:-----|:---------------|:---------------|
| Lifecycle verification | `contextdb snapshot lifecycle verify --summary PATH --report` | Gives operators one command to check the lifecycle JSON and every referenced artifact |
| Artifact consistency | Verifies backup, manifest, rehearsal, promotion receipt, and receipt-check files when present or required | Catches incomplete or mismatched backup bundles before archive, handoff, or restore |
| Promotion guard | Requires promotion and receipt-check artifacts only when `promoted` is true | Keeps non-promoted backup preflights valid without fake promotion files |
| Feature metadata | `lifecycle-summary-verify` | Lets live deployments advertise lifecycle verification through introspection |
| Test coverage | CLI helper tests cover promoted, non-promoted, and bad receipt-check summaries | Improves confidence in the backup durability workflow |

## Compatibility

This release only adds a new snapshot subcommand and feature metadata. Existing export, import, verify, rehearse, receipt verification, doctor, Norn, REST, GraphQL, and SDK behavior is unchanged.
