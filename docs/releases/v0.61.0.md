# v0.61.0 Release Recap

v0.61.0 is a non-breaking release-quality slice that makes ranking eval baselines easier to store, compare, and review across tagged releases.

## What Changed

| Area | New in v0.61.0 | Why it matters |
|:-----|:---------------|:---------------|
| Baseline artifacts | `contextdb eval ranking --baseline-dir DIR` | Writes both JSON and Markdown release baselines with the current version in the filename |
| Baseline comparison | `contextdb eval ranking --compare-baseline-dir DIR --diff-markdown` | Resolves the latest previous `ranking-eval-vX.Y.Z.json` automatically |
| Filename convention | `ranking-eval-vX.Y.Z.json` and `ranking-eval-vX.Y.Z.md` | Gives release reviews a predictable, tag-aligned ranking history |
| Feature metadata | `ranking-eval-baseline-policy` | Lets live deployments advertise support for versioned ranking baselines |

## Compatibility

This release only adds optional CLI flags and feature metadata. Existing `contextdb eval ranking`, `--out`, `--markdown-out`, `--compare`, and diff output flags are unchanged.

## Operator Flow

```bash
contextdb eval ranking --baseline-dir .contextdb/ranking-baselines
contextdb eval ranking --compare-baseline-dir .contextdb/ranking-baselines --diff-markdown
```

Use this flow when cutting a release: keep the versioned baseline artifacts, then compare the next release against the latest previous baseline in the same directory.
