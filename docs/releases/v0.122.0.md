# v0.122.0

v0.122.0 is a non-breaking release-candidate build for ContextDB claim review operations, Hermes-aware review workflows, admin review surfaces, and Norn-hosted worker deployment.

## What's New

| Area | Surface | Why it matters |
|:-----|:--------|:---------------|
| Claim review worker | `contextdb worker review` | Runs centralized claim validation, stale/resolve planning, and dry-run-first mutation from the durable ContextDB deployment instead of per app |
| Evaluators | `rules`, `webhook`, `openai`, `anthropic`, `xai`, `hybrid` | Keeps the default no-key evaluator conservative while allowing provider or webhook-assisted review when explicitly configured |
| Worker audit | Review worker run summaries | Records cycle id, evaluator mode, dry-run flag, counts, and planned decisions through Go, REST, GraphQL, TypeScript, and Python SDK surfaces |
| Admin operations | Review ops dashboard, source quarantine, graph filters, ranking local baselines | Gives operators queue posture, worker runs, source quarantine candidates, filtered graph context, and repeatable ranking comparisons in `/admin/` |
| Hermes integration | `hermes-memory-contextdb` review tools | Lets agents inspect and annotate review tasks while claim mutation remains centralized in the ContextDB worker |
| Norn deployment | `infraspec.yaml` and `scripts/norn-smoke.sh` | Deploys web plus review-worker processes and verifies web health, worker health, queue derivation, dry-run execution, and durable worker-run lookup |
| Reliability lanes | Postgres fault injection and scheduled soak CI | Adds malformed DSN, unreachable Postgres, read-only migration failure coverage, and a longer scheduled concurrency soak |

## Compatibility

This release is additive. The default review worker evaluator is `rules`, uses no provider key, and runs in dry-run mode unless operators explicitly configure execution. Existing write, retrieve, feedback, ranking, and deployment APIs remain compatible.
