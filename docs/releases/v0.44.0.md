# v0.44.0 Release Recap

v0.44.0 is a non-breaking review workflow release that adds retry candidates for failed review handoff webhook deliveries.

## What Changed

| Area | New in v0.44.0 | Why it matters |
|:-----|:---------------|:---------------|
| Go SDK retry queue | `ReviewHandoffRetryCandidates` | Lets clients find unresolved failed handoff deliveries without sending retries |
| REST retry queue | `GET /v1/namespaces/{ns}/review/handoff-webhooks/retry-candidates` | Gives operators an HTTP view of failed delivery groups by digest event and target URL |
| GraphQL retry queue | `reviewHandoffRetryCandidates(...)` | Gives debugger UIs retry review data beside receipts and delivery controls |
| Feature metadata | `review-handoff-retry-candidates` | Lets live deployments advertise read-only retry candidate support |

## Compatibility

This release only adds read-only retry candidate APIs. It does not send retries, mutate receipts, or change handoff webhook delivery behavior.

## Operator Flow

```bash
curl "http://localhost:7701/v1/namespaces/my-app/review/handoff-webhooks/retry-candidates"
```

Candidates include the digest event ID, target URL, last receipt ID, last attempt time, attempt count, last status code, payload hash, and last error. Groups with a later successful receipt are omitted.
