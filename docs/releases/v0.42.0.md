# v0.42.0 Release Recap

v0.42.0 is a non-breaking review workflow release that adds explicit synchronous execution for review handoff webhooks.

## What Changed

| Area | New in v0.42.0 | Why it matters |
|:-----|:---------------|:---------------|
| Go SDK execution | `ReviewHandoffWebhookDeliver` | Lets clients send saved handoff snapshots after explicitly opting in with `Execute: true` |
| REST execution | `POST /v1/namespaces/{ns}/review/handoff-webhooks/deliver` | Gives operators a controlled HTTP delivery path with timeout and response capture |
| GraphQL execution | `deliverReviewHandoffWebhook(...)` | Gives debugger UIs and automation tools a mutation for push-style handoff delivery |
| Feature metadata | `review-handoff-webhook-execution` | Lets live deployments advertise explicit handoff webhook execution support |

## Compatibility

This release only adds an opt-in execution path. Dry-run planning remains unchanged, and execution does not add background retries, queues, or stored delivery receipts.

## Operator Flow

```bash
curl -X POST http://localhost:7701/v1/namespaces/my-app/review/handoff-webhooks/deliver \
  -H 'Content-Type: application/json' \
  -d '{
    "owner": "alice",
    "escalation_level": "review_overdue",
    "target_url": "https://ops.example.test/contextdb/handoffs",
    "secret": "webhook-signing-secret",
    "execute": true,
    "timeout_ms": 5000
  }'
```

Each matching saved handoff is sent as one synchronous `POST`. The response records `executed`, `status_code`, `response_body`, and `error` for each delivery.
