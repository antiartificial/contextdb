# v0.35.0 Release Recap

v0.35.0 is a non-breaking operations release that adds dry-run-first Norn manifest publishing.

## What Changed

| Area | New in v0.35.0 | Why it matters |
|:-----|:---------------|:---------------|
| Norn publish | `contextdb norn publish --dry-run --report` | Lets operators validate the exact service entry and publish target before changing Norn |
| Explicit write | `contextdb norn publish --execute --publish-url URL --token TOKEN --report` | Keeps registration writes intentional and compatible with authenticated Norn endpoints |
| Publish report | JSON status, response, generated entry, and validation errors | Gives automation a durable audit trail for registration attempts |
| Feature metadata | `norn-manifest-publish` | Lets live deployments advertise dry-run-first publish support through introspection |

## Compatibility

This release only adds a new Norn publish command and feature metadata. Existing Norn manifest generation, validation, drift checks, backup lifecycle tools, REST, GraphQL, and SDK behavior is unchanged.

## Operator Flow

```bash
contextdb norn publish \
  --endpoint "$CONTEXTDB_URL" \
  --publish-url "$NORN_PUBLISH_URL" \
  --report
```

That default dry run exits zero after validating the generated entry. To perform the authenticated write, opt in explicitly:

```bash
contextdb norn publish \
  --endpoint "$CONTEXTDB_URL" \
  --publish-url "$NORN_PUBLISH_URL" \
  --token "$NORN_TOKEN" \
  --execute \
  --report
```
