# v0.72.0 Release Recap

v0.72.0 is a non-breaking documentation and release-operations release that adds a ranking baseline retention cookbook.

## What's New

| Area | New in v0.72.0 | Why it matters |
|:-----|:---------------|:---------------|
| Ranking release operations | Ranking baseline retention cookbook | Gives teams concrete keep-count recipes for tagged releases, weekly release trains, active ranking tuning, and stable maintenance |
| CI artifacts | Baseline artifact storage pattern | Shows how to keep JSON and Markdown ranking outputs available for release review instead of losing them in transient build caches |
| Retention review | Review and delete-script flow | Documents the read-only retention report first, then the reviewable `rm -- ...` deletion plan |
| Feature metadata | `ranking-eval-baseline-retention-cookbook` | Lets live deployments advertise that the cookbook is part of the supported docs surface |

## Operator Flow

Capture a versioned baseline during release review:

```bash
contextdb eval ranking --baseline-dir .contextdb/ranking-baselines --markdown
```

Compare the current run with the latest previous baseline:

```bash
contextdb eval ranking --compare-baseline-dir .contextdb/ranking-baselines --diff-markdown
```

Review retention before pruning:

```bash
contextdb eval ranking --baseline-retention-dir .contextdb/ranking-baselines --baseline-retention-keep 5
```

Generate a deletion plan only after the retained and pruneable sets look right:

```bash
contextdb eval ranking --baseline-retention-dir .contextdb/ranking-baselines --baseline-retention-keep 5 --emit-delete-script > prune-ranking-baselines.sh
```

## Compatibility

This release is intentionally non-breaking. It updates docs, release metadata, and package versions without changing CLI behavior, API schemas, storage formats, or ranking calculations.
