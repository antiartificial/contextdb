# v0.19.0 Release Recap

v0.19.0 is a non-breaking operations release that connects snapshot export with backup readiness. `contextdb snapshot export` can now write a doctor-compatible marker only after the export succeeds.

## What's New

| Area | New in v0.19.0 | Why it matters |
|:-----|:---------------|:---------------|
| Export marker | `contextdb snapshot export --backup-marker /path/.last-backup` | Records a successful backup timestamp directly from the export command |
| Doctor pairing | The marker format works with `contextdb doctor --backup-marker` | Makes backup freshness part of the existing readiness report |
| Success-only write | The marker is written after snapshot export completes | Avoids marking failed backups as fresh |
| Feature metadata | `snapshot-backup-marker` feature tag | Lets live deployments advertise backup marker support |

## Compatibility

The flag is additive. Existing snapshot export commands behave the same unless `--backup-marker` is provided.
