# v0.88.0 Release Recap

v0.88.0 is a non-breaking operations release that makes the built-in admin dashboard real on the observe port, adds a first belief debugger slice, and keeps the manifest reliability work moving with bundle index metadata.

## What's New

| Area | New in v0.88.0 | Why it matters |
|:-----|:---------------|:---------------|
| Admin dashboard | `/admin/` on the observe port | The documented dashboard is now mounted beside metrics, health, and pprof |
| Belief debugger UI | `/admin/debugger` and the debugger panel on `/admin/` | Operators can inspect a known node without writing raw API calls |
| Belief debugger API | `/admin/api/belief?ns=NS&id=UUID` | Returns source, support, contradiction, provenance, and confidence-history evidence |
| Bundle index | `ranking-baseline-manifest-verification-index.json` | Gives CI one pointer file for ranking manifest verification artifacts |
| Feature metadata | `admin-observe-dashboard`, `admin-belief-debugger`, `ranking-baseline-bundle-index-metadata` | Lets live deployments advertise the dashboard/debugger and bundle index support |

## Operator Flow

Open the dashboard on the observe port:

```bash
open http://localhost:7702/admin/
```

Inspect one claim by namespace and node ID:

```bash
curl "http://localhost:7702/admin/api/belief?ns=my-app&id=NODE_UUID"
```

Write a ranking manifest verification bundle with index metadata:

```bash
contextdb eval ranking baseline manifest verify \
  --manifest ranking-baseline-manifest.json \
  --bundle-dir ranking-baseline-verification
```

The bundle now includes JSON, Markdown, annotation, and index JSON artifacts.

## Compatibility

This release is intentionally non-breaking. REST, GraphQL, SDK, and existing CLI verification behavior are unchanged.
