# v0.46.0 Release Recap

v0.46.0 is a non-breaking review workflow release that adds read-only retry backoff recommendations for failed review handoff webhook deliveries.

## What Changed

| Area | New in v0.46.0 | Why it matters |
|:-----|:---------------|:---------------|
| Go SDK retry recommendations | `ReviewHandoffRetryRecommendations` | Lets clients see recommended retry timing without sending a retry |
| REST retry recommendations | `GET /v1/namespaces/{ns}/review/handoff-webhooks/retry-recommendations` | Gives operators a pacing view before they execute a retry |
| GraphQL retry recommendations | `reviewHandoffRetryRecommendations(...)` | Lets inspector UIs show readiness, delay, and reason beside retry candidates |
| Feature metadata | `review-handoff-retry-backoff` | Lets live deployments advertise read-only retry pacing support |

## Compatibility

This release only adds read-only recommendation APIs. It does not schedule background retries, change webhook payloads, or alter explicit retry execution.

## Operator Flow

List retry recommendations:

```bash
curl "http://localhost:7701/v1/namespaces/my-app/review/handoff-webhooks/retry-recommendations"
```

Each recommendation includes candidate identity, attempt count, last failure details, `recommended_after`, `delay_seconds`, `ready`, and `reason`. The first failed attempt recommends a 60 second wait; later attempts use capped exponential backoff.
