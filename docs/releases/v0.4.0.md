# v0.4.0 Release Recap

v0.4.0 is a non-breaking release focused on making live contextdb deployments self-describing. Clients and operators can now discover the running version, supported feature surface, and embedded Postgres schema migrations through REST and GraphQL.

## What Changed

| Area | New in v0.4.0 | Why it matters |
|:-----|:--------------|:---------------|
| Version introspection | `GET /v1/version` and GraphQL `version` | Lets clients confirm API/docs compatibility against a live server |
| Feature discovery | `GET /v1/features` and GraphQL `features` | Makes documented feature tags machine-readable |
| Migration discovery | `GET /v1/migrations` and GraphQL `migrations` | Exposes the embedded Postgres migration set without shell access |
| Doctor command | `contextdb doctor --url http://host:7701` | Provides a CLI health report for live REST deployments without mutating data |
| Durability tests | Badger restart coverage for nodes, vectors, history, feedback, and dedup | Protects the embedded deployment mode used by local and Mini/Norn installs |
| Ranking golden tests | Belief-system and agent-memory ranking fixtures | Catches unintentional scoring drift across namespace modes |
| API contract tests | gRPC write/retrieve/feedback coverage plus REST failure-path coverage | Keeps public API behavior aligned across transports |

## Non-Breaking Behavior

The release adds endpoints, GraphQL fields, package metadata, and tests. Existing SDK, REST, gRPC, and GraphQL callers do not need to change.

## Feature Tags

| Tag | Meaning |
|:----|:--------|
| `v0.4` | Introduced or substantially expanded in v0.4.0 |
| `non-breaking` | Does not require existing clients to change |
| `introspection` | Helps software inspect the running server and supported capabilities |
| `durability` | Protects restart, persistence, and store consistency behavior |
| `ranking` | Covers score ordering, score components, or retrieval quality |
