# v0.41.0 Release Recap

v0.41.0 is a non-breaking review workflow release that adds signed dry-run webhook plans for saved review handoffs.

## What Changed

| Area | New in v0.41.0 | Why it matters |
|:-----|:---------------|:---------------|
| Go SDK planner | `ReviewHandoffWebhookPlan` | Lets clients preview webhook payloads, signatures, headers, and retry metadata before any delivery engine exists |
| REST planner | `POST /v1/namespaces/{ns}/review/handoff-webhooks/plan` | Gives operators a safe HTTP workflow for validating handoff delivery payloads |
| GraphQL planner | `reviewHandoffWebhookPlan(...)` | Gives debugger UIs a dry-run delivery view beside saved handoff feeds |
| Feature metadata | `review-handoff-webhook-plan` | Lets live deployments advertise signed dry-run webhook planning support |

## Compatibility

This release only adds planning APIs. It does not send outbound requests, store delivery attempts, or change existing review queue, digest, export, or handoff feed behavior.

## Operator Flow

```bash
curl -X POST http://localhost:7701/v1/namespaces/my-app/review/handoff-webhooks/plan \
  -H 'Content-Type: application/json' \
  -d '{
    "owner": "alice",
    "escalation_level": "review_overdue",
    "target_url": "https://ops.example.test/contextdb/handoffs",
    "secret": "webhook-signing-secret"
  }'
```

The response includes `dry_run: true`, the JSON payload, `payload_sha256`, optional `sha256=` HMAC signature, delivery headers, and retry metadata.
