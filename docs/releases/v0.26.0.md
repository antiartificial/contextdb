# v0.26.0 Release Recap

v0.26.0 is a non-breaking restore audit release. Snapshot imports can now write a promotion receipt after a successful restore.

## What's New

| Area | New in v0.26.0 | Why it matters |
|:-----|:---------------|:---------------|
| Promotion receipt | `contextdb snapshot import --promotion-report promotion.json` | Leaves a durable JSON artifact after a real import succeeds |
| Operator note | `--promotion-note "..."` | Carries human approval context into the receipt |
| Import counts | Full `SnapshotReport` embedded in the receipt | Records nodes, edges, vectors, namespace overrides, and diff counts |
| Promotion metadata | `promoted_at`, `namespace`, `backup_file`, `contextdb_version` | Makes restore events auditable after the fact |
| Feature metadata | `restore-promotion-receipt` | Lets deployments advertise support through introspection |

## Compatibility

The receipt flags are optional. Dry-run imports still do not write promotion receipts, and existing export, verify, rehearse, import, manifest, marker, and doctor workflows keep their behavior unchanged.
