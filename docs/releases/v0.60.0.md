# v0.60.0 Release Recap

v0.60.0 is a non-breaking review operations release that adds scoped retry fatigue views.

## What Changed

| Area | New in v0.60.0 | Why it matters |
|:-----|:---------------|:---------------|
| REST filters | `retry-fatigue?owner=alice&escalation_level=review_overdue` | Lets operators focus endpoint fatigue on one workload owner or escalation class |
| GraphQL filters | `reviewHandoffRetryFatigue(owner:, escalationLevel:)` | Gives inspector UIs the same scoped fatigue view |
| Go SDK filters | `ReviewHandoffRetryFatigueFiltered` | Lets tools request scoped fatigue summaries without post-processing |
| Feature metadata | `review-handoff-retry-fatigue-filters` | Lets live deployments advertise owner-aware retry fatigue filtering |

## Compatibility

This release only adds optional filters and a new Go helper. Existing unfiltered retry fatigue responses, Markdown exports, retry recommendations, retry execution, and delivery receipt behavior are unchanged.

## Operator Flow

```bash
curl "http://localhost:7701/v1/namespaces/my-app/review/handoff-webhooks/retry-fatigue?owner=alice&escalation_level=review_overdue"
```

Use these filters when incident handoffs need a focused view for a single owner or escalation lane.
