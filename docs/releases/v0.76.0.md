# v0.76.0 Release Recap

v0.76.0 is a non-breaking ranking operations release that adds a machine-readable artifact manifest for versioned ranking baselines.

## What's New

| Area | New in v0.76.0 | Why it matters |
|:-----|:---------------|:---------------|
| Ranking retention | `--baseline-manifest-out PATH` | Writes a JSON inventory for all retained and pruneable baseline artifacts |
| Artifact evidence | Byte sizes and SHA-256 hashes | Gives CI a durable record of exactly which ranking JSON and Markdown baselines were reviewed |
| Missing artifacts | `missing: true` entries | Makes incomplete JSON/Markdown baseline pairs visible to release jobs |
| Feature metadata | `ranking-eval-baseline-artifact-manifest` | Lets live deployments advertise ranking baseline manifest support |

## Operator Flow

Review ranking baseline retention and write an artifact manifest:

```bash
contextdb eval ranking \
  --baseline-retention-dir .contextdb/ranking-baselines \
  --baseline-retention-keep 5 \
  --baseline-manifest-out ranking-baseline-manifest.json
```

The command remains read-only for baseline artifacts. The manifest includes each baseline artifact's version, retention status, path, existence, byte size, and SHA-256 hash.

## Compatibility

This release is intentionally non-breaking. Existing ranking eval snapshots, baseline files, retention reports, and delete-script output are unchanged unless `--baseline-manifest-out` is supplied.
