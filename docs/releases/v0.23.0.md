# v0.23.0 Release Recap

v0.23.0 is a non-breaking backup verification release. Snapshot manifests can now be checked before restore with a dedicated CLI command.

## What's New

| Area | New in v0.23.0 | Why it matters |
|:-----|:---------------|:---------------|
| CLI verifier | `contextdb snapshot verify --manifest backup.manifest.json --in backup.ndjson` | Gives operators a fast pre-restore artifact check |
| Default file resolution | `--in` can be omitted when the backup sits beside the manifest under `backup_file` | Keeps scheduled backup scripts compact |
| Verification report | `--report` prints expected and actual bytes, checksums, and record counts | Makes CI, launchd, systemd, and Norn-adjacent automation scriptable |
| Failure behavior | Mismatches exit non-zero | Prevents accidental restore from truncated or mismatched artifacts |
| Feature metadata | `backup-manifest-verify` | Lets deployments advertise support through introspection |

## Compatibility

The verifier is additive. Existing snapshot export/import flows, artifact manifests, backup markers, and restore previews keep their behavior unchanged.
