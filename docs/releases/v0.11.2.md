# v0.11.2 Release Recap

v0.11.2 is a non-breaking operations documentation release. It adds a release health page that records the verification gates used for tagged releases and links that page into the public docs.

## What Changed

| Area | New in v0.11.2 | Why it matters |
|:-----|:---------------|:---------------|
| Release health page | `docs/release-health.md` | Makes unit, docs-build, ranking, durability, API contract, and race/soak expectations visible in one place |
| Docs navigation | Release health is linked from the sidebar | Keeps GitHub Pages users one click from the release confidence summary |
| Feature metadata | `release-health-page` feature tag | Lets live deployments advertise the release-health documentation surface through introspection |
| Release process | v0.11.2 recap and release index updates | Keeps latest-first docs aligned with the tagged version |

## Non-Breaking Behavior

No runtime APIs changed in this release. The feature metadata list is additive, and existing clients can ignore the new tag.
