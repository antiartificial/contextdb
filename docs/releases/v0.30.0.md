# v0.30.0 Release Recap

v0.30.0 is a non-breaking operations release that adds dry-run retention reporting for backup lifecycle bundles.

## What's New

| Area | New in v0.30.0 | Why it matters |
|:-----|:---------------|:---------------|
| Retention report | `contextdb snapshot lifecycle retention --dir PATH --keep N --report` | Shows which lifecycle bundles are newest enough to keep and which older bundles are pruneable |
| Artifact grouping | Each bundle lists summary, backup, manifest, rehearsal, promotion, and receipt-check artifact paths with existence and byte counts | Gives operators an audit-friendly view before cleanup |
| Namespace focus | `--namespace NAME` narrows the report to one namespace | Makes shared backup directories easier to review |
| Dry-run behavior | The command never deletes files | Keeps retention planning non-destructive |
| Feature metadata | `lifecycle-retention-report` | Lets live deployments advertise retention reporting through introspection |

## Compatibility

This release only adds a new snapshot lifecycle subcommand and feature metadata. Existing lifecycle verification, backup, restore, doctor, Norn, REST, GraphQL, and SDK behavior is unchanged.
