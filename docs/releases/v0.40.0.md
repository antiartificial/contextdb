# v0.40.0 Release Recap

v0.40.0 is a non-breaking review workflow release that adds polling-friendly handoff feeds for saved escalation digests.

## What Changed

| Area | New in v0.40.0 | Why it matters |
|:-----|:---------------|:---------------|
| Go SDK feed | `ReviewHandoffs` | Lets clients poll saved escalation digest snapshots filtered by owner and escalation level |
| REST feed | `GET /v1/namespaces/{ns}/review/handoffs` | Gives owners and dashboards a narrow handoff feed without fetching every saved digest |
| GraphQL feed | `reviewHandoffs(...)` | Gives debugger UIs filtered handoff snapshots beside digest export APIs |
| Feature metadata | `review-handoff-feed` | Lets live deployments advertise polling-friendly handoff support |

## Compatibility

This release only adds filtered feed APIs over existing saved escalation digest snapshots. Existing review queue, escalation digest, and digest export behavior is unchanged.

## Operator Flow

```bash
curl "http://localhost:7701/v1/namespaces/my-app/review/handoffs?owner=alice&escalation_level=review_overdue"
```

The feed returns saved digest snapshots with only the groups matching the requested owner and escalation level.
