# v0.92.0 Release Recap

v0.92.0 is a non-breaking dashboard release that moves the admin metrics and debugger surface into an embedded Svelte app shell.

## What's New

| Area | New in v0.92.0 | Why it matters |
|:-----|:---------------|:---------------|
| Svelte admin shell | `/admin/` is served from a built Svelte bundle embedded in the Go binary | Gives the dashboard a typed component foundation for richer metrics and debugger workflows |
| Build path | `npm run admin:build` | Rebuilds the admin app into `internal/admin/dist` for Go embedding |
| Debugger continuity | Metrics, search, and belief inspection still call the existing admin APIs | Keeps the current operator workflow stable while the UI architecture changes |
| Feature metadata | `admin-svelte-shell` | Lets live deployments advertise the componentized admin shell |

## Operator Flow

Open the dashboard as before:

```bash
open http://localhost:7702/admin/
```

Rebuild the embedded admin app after frontend changes:

```bash
npm run admin:build
go test -count=1 ./internal/admin
```

The Svelte app continues to use:

```bash
curl http://localhost:7702/admin/api/metrics
curl "http://localhost:7702/admin/api/search?ns=my-app&q=auth&limit=10"
curl "http://localhost:7702/admin/api/belief?ns=my-app&id=NODE_UUID"
```

## Compatibility

This release is intentionally non-breaking. `/admin/`, `/admin/debugger`, `/admin/api/metrics`, `/admin/api/search`, and `/admin/api/belief` keep the same route shape.

## Verification
