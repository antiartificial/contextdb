# v0.20.0 Release Recap

v0.20.0 is a non-breaking restore inspectability release. Snapshot dry-run and import reports now classify node records as new, changed, or unchanged, so operators can preview restore impact before writing.

## What's New

| Area | New in v0.20.0 | Why it matters |
|:-----|:---------------|:---------------|
| Go client reports | `SnapshotReport.NewNodes`, `ChangedNodes`, and `UnchangedNodes` | Shows restore impact without parsing snapshot records manually |
| Dry-run diff preview | `ValidateSnapshotReport` compares snapshot nodes against the target namespace without writing | Lets operators preview whether an import would add, update, or leave nodes unchanged |
| CLI reports | `contextdb snapshot import --dry-run --report` includes node diff counts | Makes restore previews scriptable for backup checks and operational runbooks |
| Feature metadata | `snapshot-diff-preview` | Lets deployments advertise support through version and feature introspection |

## Compatibility

All fields are additive. Existing import and validation helpers keep their behavior, quiet CLI output is unchanged unless `--report` is requested, and this remains a non-breaking pre-1.0 minor release.
