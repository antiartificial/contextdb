# v0.12.0 Release Recap

v0.12.0 is a non-breaking workflow release that adds durable review decisions. The review queue remains derived from feedback, low-confidence claims, and contradictions, while assignment, snooze, resolution, owner, decision, and notes are now stored as append-only workflow events.

## What Changed

| Area | New in v0.12.0 | Why it matters |
|:-----|:---------------|:---------------|
| Go SDK workflow | `RecordReviewDecision` and `ReviewDecisions` | Lets applications persist triage decisions around derived review tasks |
| REST workflow | `GET` and `POST /v1/namespaces/{ns}/review/decisions` | Gives operators an auditable HTTP workflow surface |
| GraphQL workflow | `recordReviewDecision` mutation and `reviewDecisions` query | Gives debugger UIs durable state without making queue generation stateful |
| Queue overlay | `ReviewItem` includes latest status, owner, decision, note, recheck time, and reviewed time | Lets active queues reflect assignment and snooze state |
| Feature metadata | `review-workflow-persistence` feature tag | Lets live deployments advertise durable review workflow support |

## Non-Breaking Behavior

All fields and endpoints are additive. Existing review queue clients continue to work, while clients that read the new fields can hide resolved items and defer snoozed items until their re-check time.
