# v0.25.0 Release Recap

v0.25.0 is a non-breaking restore operations release. Rehearsal reports now include promotion checklist metadata so a successful preflight can hand operators the next command clearly.

## What's New

| Area | New in v0.25.0 | Why it matters |
|:-----|:---------------|:---------------|
| Rehearsal timestamp | `rehearsed_at` | Records when the preflight was run |
| Target namespace | `target_namespace` | Makes the intended restore target explicit in automation logs |
| Promotion command | `recommended_import_command` | Provides a shell-quoted `contextdb snapshot import --namespace ... --in ... --report` command |
| Safer scripting | Quoted namespace and file path values | Handles paths or namespace names with spaces and quotes |
| Feature metadata | `restore-promotion-checklist` | Lets deployments advertise support through introspection |

## Compatibility

The new rehearsal fields are additive. Existing export, verify, rehearse, import, manifest, marker, and doctor workflows keep their behavior unchanged.
