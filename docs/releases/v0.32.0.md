# v0.32.0 Release Recap

v0.32.0 is a non-breaking operations release that adds compact lifecycle manifest indexes for backup directories.

## What's New

| Area | New in v0.32.0 | Why it matters |
|:-----|:---------------|:---------------|
| Manifest index | `contextdb snapshot lifecycle index --dir PATH --out PATH --report` | Writes one portable JSON catalog for lifecycle backup bundles |
| Artifact hashes | Indexed artifacts include byte counts and SHA-256 hashes | Makes copied backup catalogs easier to audit |
| Retention context | Indexes include keep/pruneable decisions and delete-plan commands | Preserves the retention review beside the artifacts |
| Default path | Omitting `--out` writes `contextdb-backups.index.json` in `--dir` | Gives operators a predictable catalog name |
| Feature metadata | `lifecycle-manifest-index` | Lets live deployments advertise manifest-index support through introspection |

## Compatibility

This release only adds a new lifecycle subcommand and feature metadata. Existing lifecycle reports, deletion plans, lifecycle verification, backup, restore, doctor, Norn, REST, GraphQL, and SDK behavior is unchanged.
