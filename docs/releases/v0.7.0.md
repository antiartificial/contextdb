# v0.7.0 Release Recap

v0.7.0 is a non-breaking release that adds a claim review queue. The queue turns existing epistemic signals, including refutations, stale feedback, low confidence, and contradiction edges, into ranked operator tasks.

## What Changed

| Area | New in v0.7.0 | Why it matters |
|:-----|:--------------|:---------------|
| Go SDK review queue | `NamespaceHandle.ReviewQueue(ctx, req)` | Lets applications surface claims that need human or agent review |
| REST review queue | `GET /v1/namespaces/{ns}/review/queue` | Gives operators a simple endpoint for moderation and cleanup tasks |
| GraphQL review queue | `reviewQueue(namespace:, after:, lowConfidenceThreshold:, limit:)` | Gives debugger UIs a graph-friendly task list |
| Feature metadata | `claim-review-queue` feature tag | Lets live deployments advertise queue support through introspection |

## Non-Breaking Behavior

The queue is additive and derived from existing data. It does not mutate claims, change retrieval ranking, or alter feedback behavior.
