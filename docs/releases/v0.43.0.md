# v0.43.0 Release Recap

v0.43.0 is a non-breaking review workflow release that adds append-only delivery receipts for executed review handoff webhooks.

## What Changed

| Area | New in v0.43.0 | Why it matters |
|:-----|:---------------|:---------------|
| Go SDK receipts | `ReviewHandoffDeliveryReceipts` | Lets clients audit webhook delivery outcomes after execution |
| REST receipts | `GET /v1/namespaces/{ns}/review/handoff-webhooks/receipts` | Gives operators a durable HTTP view of delivery status, payload hashes, response hashes, and errors |
| GraphQL receipts | `reviewHandoffDeliveryReceipts(...)` | Gives debugger UIs and automation tools receipt history beside handoff delivery controls |
| Event log | `review_handoff_delivery_receipt` | Keeps delivery evidence append-only and replayable |
| Feature metadata | `review-handoff-delivery-receipts` | Lets live deployments advertise delivery receipt support |

## Compatibility

This release only adds receipt events and list APIs. Existing dry-run plans and explicit webhook execution behavior remain unchanged.

## Operator Flow

```bash
curl "http://localhost:7701/v1/namespaces/my-app/review/handoff-webhooks/receipts?after=2026-05-31T00:00:00Z"
```

Receipts include the digest event ID, target URL, delivery timestamp, success flag, status code, payload hash, response hash, and error text when delivery fails.
