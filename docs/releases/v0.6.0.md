# v0.6.0 Release Recap

v0.6.0 is a non-breaking release that turns feedback audit records into source trust timelines. The timeline shows when feedback changed a source's effective credibility, making moderation and agent-trust reviews easier to explain.

## What Changed

| Area | New in v0.6.0 | Why it matters |
|:-----|:--------------|:---------------|
| Go SDK timeline | `NamespaceHandle.SourceTrustTimeline(ctx, sourceID, after)` | Lets applications inspect trust changes for one source |
| REST timeline | `GET /v1/namespaces/{ns}/sources/{sourceID}/trust` | Gives operators a simple source-trust audit endpoint |
| GraphQL timeline | `sourceTrustTimeline(namespace:, sourceId:, after:)` | Gives debugger UIs a graph-friendly trust history |
| Feature metadata | `source-trust-timeline` feature tag | Lets live deployments advertise timeline support through introspection |

## Non-Breaking Behavior

The timeline is additive. It is derived from feedback events, so existing feedback behavior and source updates are unchanged.
