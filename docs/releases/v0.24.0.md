# v0.24.0 Release Recap

v0.24.0 is a non-breaking restore preflight release. `contextdb snapshot rehearse` combines artifact verification with a dry-run restore report so operators can check a backup in one command before promotion.

## What's New

| Area | New in v0.24.0 | Why it matters |
|:-----|:---------------|:---------------|
| Rehearsal command | `contextdb snapshot rehearse --manifest backup.manifest.json --in backup.ndjson --namespace restore-preview` | Runs checksum/size/count verification and dry-run import validation together |
| Combined report | `--report` includes `verification` and `restore` sections | Gives automation one JSON result for backup preflight |
| Default backup resolution | `--in` can be omitted when the manifest's `backup_file` is beside the manifest | Keeps scheduled backup scripts simple |
| Failure behavior | Verification or dry-run failures exit non-zero | Prevents restore promotion from untrusted artifacts |
| Feature metadata | `restore-rehearsal` | Lets deployments advertise support through introspection |

## Compatibility

The rehearsal command is additive. Existing export, verify, import, manifest, marker, and doctor workflows keep their behavior unchanged.
