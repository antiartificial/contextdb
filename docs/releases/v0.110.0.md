# v0.110.0

v0.110.0 is a non-breaking policy release that lets one centralized worker use different review behavior per namespace.

## What's New

| Area | Surface | Why it matters |
|:-----|:--------|:---------------|
| Namespace policy | Review worker configuration | Supports namespace-specific dry-run rollout, thresholds, owner routing, evaluator mode, review types, and allowed actions |
| Agent memory safety | Hermes and application namespaces | Lets high-sensitivity memory namespaces use tighter validation and mutation rules than ordinary application data |
| Operational rollout | Centralized worker policy map | Keeps worker deployment shared while preserving per-namespace governance |

## Verification

- `go test ./...`
- `npm run docs:build`
- Review worker policy coverage for namespace-specific configuration

## Compatibility

This release is additive. Existing workers keep their default policy unless operators configure namespace-specific overrides.
