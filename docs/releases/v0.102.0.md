# v0.102.0 Release Recap

v0.102.0 is a non-breaking reliability release that publishes derived KV refresh receipt fixtures for integration and CI consumers.

## What's New

| Area | New in v0.102.0 | Why it matters |
|:-----|:----------------|:---------------|
| Passing fixture | `/fixtures/kv-refresh/valid-receipt.json` and `/fixtures/kv-refresh/valid-value.json` | Gives clients a known-good receipt/value pair for verifier wiring |
| Failing fixture | `/fixtures/kv-refresh/mismatched-value.json` | Lets integrations confirm drift failures are surfaced correctly |
| Fixture coverage | Public fixture verifier tests | Keeps the docs fixtures aligned with `verifyKVRefreshReceipt` behavior |
| Docs recipe | KV derivation fixture section | Shows copyable CLI commands for local and CI checks |
| Feature metadata | `kv-refresh-receipt-fixtures` | Lets live deployments advertise fixture availability |

## Operator Flow

Use the passing fixture to test verifier plumbing:

```bash
contextdb repair kv-cache receipt verify \
  --receipt docs/public/fixtures/kv-refresh/valid-receipt.json \
  --value-file docs/public/fixtures/kv-refresh/valid-value.json \
  --report
```

Use the mismatched value fixture when CI needs to assert that drift is rejected.

## Compatibility

This release is intentionally non-breaking. Runtime behavior is unchanged; the release publishes docs fixtures and adds test coverage around them.
