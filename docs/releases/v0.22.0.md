# v0.22.0 Release Recap

v0.22.0 is a non-breaking backup auditability release. Snapshot export can now write a JSON artifact manifest next to the NDJSON backup, giving operators a checksummed sidecar for off-host copies and restore previews.

## What's New

| Area | New in v0.22.0 | Why it matters |
|:-----|:---------------|:---------------|
| CLI export manifest | `contextdb snapshot export --manifest backup.manifest.json` | Writes a machine-readable sidecar after a successful export |
| Checksum and size | SHA-256 checksum plus backup byte count | Helps detect truncated, stale, or mismatched backup files before restore |
| Record counts | Line, node, edge, and source counts | Gives scripts quick sanity checks without re-importing the backup |
| Version metadata | Contextdb version, namespace, backup filename, created time, and marker path | Makes copied artifacts easier to audit later |
| Feature metadata | `backup-artifact-manifest` | Lets deployments advertise support through introspection |

## Compatibility

The new `--manifest` flag is optional. Existing snapshot exports, backup markers, restore previews, and doctor checks keep their behavior unchanged.
