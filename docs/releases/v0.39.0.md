# v0.39.0 Release Recap

v0.39.0 is a non-breaking review workflow release that adds durable escalation digest snapshots for review handoffs.

## What Changed

| Area | New in v0.39.0 | Why it matters |
|:-----|:---------------|:---------------|
| Go SDK export | `RecordReviewEscalationDigest` and `ReviewEscalationDigests` | Lets applications save and list escalation handoff snapshots |
| REST export | `POST` and `GET /v1/namespaces/{ns}/review/escalation-digests` | Gives operators durable handoff records through HTTP |
| GraphQL export | `recordReviewEscalationDigest` and `reviewEscalationDigests` | Gives debugger UIs a mutation/query pair for saved digest snapshots |
| Event log | `review_escalation_digest` events | Keeps digest handoffs append-only and auditable |
| Feature metadata | `review-escalation-digest-export` | Lets live deployments advertise saved digest snapshot support |

## Compatibility

This release only adds new export/list APIs and an append-only event type. Existing review queue, escalation metadata, and digest query behavior is unchanged.

## Operator Flow

```bash
curl -X POST http://localhost:7701/v1/namespaces/my-app/review/escalation-digests \
  -H 'Content-Type: application/json' \
  -d '{"escalation_after_hours":72,"note":"weekly review handoff"}'
```

List saved snapshots:

```bash
curl "http://localhost:7701/v1/namespaces/my-app/review/escalation-digests"
```
