# v0.53.0 Release Recap

v0.53.0 is a non-breaking review operations release that adds Markdown export for retry fatigue incident handoffs.

## What Changed

| Area | New in v0.53.0 | Why it matters |
|:-----|:---------------|:---------------|
| Go SDK export | `ReviewHandoffRetryFatigueMarkdown` | Renders endpoint fatigue summaries as human-readable notes |
| REST export | `retry-fatigue?format=markdown` | Lets operators pipe retry fatigue directly into incident handoffs |
| Markdown contents | Top endpoint, readiness counts, status families, latest failure | Makes repeated webhook failures scannable without JSON tooling |
| Feature metadata | `review-handoff-retry-fatigue-markdown` | Lets live deployments advertise Markdown handoff export support |

## Compatibility

This release adds a Markdown rendering path only. Existing JSON retry fatigue responses, retry recommendations, retry execution, and receipt storage are unchanged.

## Operator Flow

```bash
curl "http://localhost:7701/v1/namespaces/my-app/review/handoff-webhooks/retry-fatigue?format=markdown"
```

The Markdown output starts with the top failing endpoint and then includes a table of candidates, attempts, ready and waiting counts, status families, and latest failure details.
