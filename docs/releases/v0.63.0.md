# v0.63.0 Release Recap

v0.63.0 is a non-breaking durability release that lets KV cache repair derive reviewed session context values from graph data.

## What Changed

| Area | New in v0.63.0 | Why it matters |
|:-----|:---------------|:---------------|
| Derived KV value | `contextdb repair kv-cache --derive recent-nodes` | Builds a compact recent-node session context payload from valid graph nodes |
| Derivation scope | `--derive-namespace`, `--derive-label`, and `--derive-limit` | Lets operators control which graph facts feed the reviewed cache value |
| Dry-run review | Existing `--report` and `--execute` behavior is preserved | Keeps generated values inspectable before they are written to KV |
| Feature metadata | `kv-cache-derived-recent-nodes` | Lets live deployments advertise support for graph-derived KV refresh values |

## Compatibility

This release only adds optional `contextdb repair kv-cache` flags and feature metadata. Existing literal values, value-file refreshes, dry-run reports, overwrite behavior, TTLs, and execute flow are unchanged.

## Operator Flow

```bash
contextdb repair kv-cache \
  --key context:prod:recent-session \
  --derive recent-nodes \
  --derive-namespace prod \
  --derive-label SessionContext \
  --derive-limit 5 \
  --report
```

Review the generated JSON payload first, then add `--execute` when it is ready to write.
