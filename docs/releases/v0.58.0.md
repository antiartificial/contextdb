# v0.58.0 Release Recap

v0.58.0 is a non-breaking review operations release that adds owner and escalation breakdowns to retry fatigue summaries.

## What Changed

| Area | New in v0.58.0 | Why it matters |
|:-----|:---------------|:---------------|
| Owner grouping | `owners` in retry fatigue summaries | Shows which review owners are tied to unresolved retry pressure |
| Escalation grouping | `escalation_levels` / `escalationLevels` | Separates review-overdue, source-anomaly, and unclassified handoff pressure |
| Markdown handoffs | Retry fatigue Markdown includes Owners and Escalations columns | Makes incident notes more actionable without JSON tooling |
| Feature metadata | `review-handoff-retry-fatigue-owner-groups` | Lets live deployments advertise owner-aware retry fatigue support |

## Compatibility

This release only adds fields to existing retry fatigue JSON, GraphQL, and Markdown outputs. Existing retry recommendations, retry execution, delivery receipts, and webhook behavior are unchanged.

## Operator Flow

```bash
curl "http://localhost:7701/v1/namespaces/my-app/review/handoff-webhooks/retry-fatigue?format=markdown"
```

The Markdown table now includes owner and escalation-level counts beside endpoint readiness and status-family counts.
