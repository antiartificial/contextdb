# v0.106.0

v0.106.0 is a non-breaking integration release that adds first-party provider adapters for the acquisition connector workflow.

## What Changed

| Area | Change | Why it matters |
|:-----|:-------|:---------------|
| Connector server | `contextdb connectors serve` | Runs acquisition connector endpoints outside the main contextdb API |
| OpenAI adapter | `POST /openai/search` and `/openai/crawl` | Lets acquisition execution call OpenAI web-search-backed research |
| xAI adapter | `POST /xai/search` and `/xai/crawl` | Lets acquisition execution call xAI web-search-backed research |
| Anthropic adapter | `POST /anthropic/search` and `/anthropic/crawl` | Lets acquisition execution call Claude web-search-backed research |
| Normalization | Provider responses become `{ "items": [...] }` | Keeps provider details behind the same source-constrained acquisition contract |
| Feature metadata | `acquisition-provider-connectors` | Lets live services advertise bundled provider adapter support |

## Run It

Start the connector server:

```bash
OPENAI_API_KEY=... \
XAI_API_KEY=... \
ANTHROPIC_API_KEY=... \
contextdb connectors serve \
  --addr :7780 \
  --providers openai,xai,anthropic \
  --allowed-domains docs.example.com,github.com
```

Then use the adapter endpoint from acquisition execution:

```json
{
  "budget": 2,
  "max_results": 3,
  "allowed_source_ids": ["openai:web"],
  "connectors": [
    {
      "id": "openai-web",
      "type": "search",
      "endpoint": "http://localhost:7780/openai/search",
      "allowed_source_ids": ["openai:web"],
      "default_labels": ["acquired", "provider:openai"]
    }
  ]
}
```

Dry-run behavior is unchanged: contextdb will not call the adapter until acquisition execution is run with `execute: true`.

## Configuration

| Variable | Purpose |
|:---------|:--------|
| `OPENAI_API_KEY` | OpenAI adapter API key |
| `XAI_API_KEY` | xAI adapter API key |
| `ANTHROPIC_API_KEY` | Anthropic adapter API key |
| `CONTEXTDB_OPENAI_CONNECTOR_MODEL` | OpenAI model override |
| `CONTEXTDB_XAI_CONNECTOR_MODEL` | xAI model override |
| `CONTEXTDB_ANTHROPIC_CONNECTOR_MODEL` | Anthropic model override |
| `CONTEXTDB_OPENAI_BASE_URL` | OpenAI-compatible base URL override |
| `CONTEXTDB_XAI_BASE_URL` | xAI base URL override |
| `CONTEXTDB_ANTHROPIC_BASE_URL` | Anthropic base URL override |
| `CONTEXTDB_CONNECTOR_ALLOWED_DOMAINS` | Comma-separated provider search domain allow-list |
| `CONTEXTDB_CONNECTOR_BLOCKED_DOMAINS` | Comma-separated provider search domain block-list |

Domain allow/block lists are mutually exclusive for one connector server process.

## Compatibility

This release is additive. The main contextdb REST API, acquisition execution contract, SDKs, and existing custom connector endpoints continue to work unchanged.

## Verification

The release includes provider adapter tests for OpenAI-compatible request shape, Anthropic web-search message shape, normalized connector items, and connector server routing.
