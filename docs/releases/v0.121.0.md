# v0.121.0

v0.121.0 is a non-breaking durability release that adds a scheduled extended soak lane for storage and retrieval concurrency.

## What's New

| Area | Surface | Why it matters |
|:-----|:--------|:---------------|
| Scheduled soak | GitHub Actions scheduled and manual runs | Executes a longer configurable storage/retrieval concurrency soak outside ordinary PR checks |
| Durability config | `CONTEXTDB_SOAK_DURATION`, writers, and readers | Lets CI tune soak intensity without changing test code |
| Release confidence | Extended concurrency evidence | Adds race-sensitive durability signal before release candidate promotion |

## Verification

- `go test ./...`
- `CONTEXTDB_SOAK_DURATION=1s CONTEXTDB_SOAK_WRITERS=8 CONTEXTDB_SOAK_READERS=4 go test -count=1 ./pkg/client -run TestNamespace_ConcurrentWriteRetrieveSoak`
- `npm run docs:build`

## Compatibility

This release is additive. The extended soak lane runs only in scheduled or manually dispatched CI contexts and does not slow ordinary PR checks.
