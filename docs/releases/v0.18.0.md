# v0.18.0 Release Recap

v0.18.0 is a non-breaking operations release that makes snapshot restores inspectable. Dry-run validation and real imports can now return the same structured report with processed lines, record counts, vector reindex counts, and namespace overrides.

## What's New

| Area | New in v0.18.0 | Why it matters |
|:-----|:---------------|:---------------|
| Go client reports | `ValidateSnapshotReport` and `ImportSnapshotReport` | Lets applications preview and audit restore work without parsing logs |
| CLI report flag | `contextdb snapshot import --dry-run --report` and `--report` on real imports | Gives operators machine-readable restore summaries |
| Record counts | Lines, nodes, edges, sources, and vectors | Makes backup contents and vector reindexing visible before promotion |
| Namespace override count | `namespace_overrides` | Shows when a backup is being restored into a different namespace |
| Feature metadata | `snapshot-restore-report` feature tag | Lets live deployments advertise restore report support |

## Compatibility

Existing `ImportSnapshot`, `ValidateSnapshot`, and quiet CLI import behavior are unchanged. Reports are opt-in through the new Go helper methods or the CLI `--report` flag.
