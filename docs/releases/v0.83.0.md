# v0.83.0 Release Recap

v0.83.0 is a non-breaking KV audit release that adds receipts for executed graph-derived KV refreshes.

## What's New

| Area | New in v0.83.0 | Why it matters |
|:-----|:---------------|:---------------|
| Derived KV receipt | `contextdb repair kv-cache --derive recent-nodes --execute --receipt-out PATH` | Records durable evidence after a reviewed derived cache refresh |
| Value integrity | SHA-256 hash of the reviewed derived value | Lets incident records prove which payload was written |
| Doctor follow-up | `recommended_doctor_command` in the receipt | Gives operators a copyable confirmation command for refreshed derived keys |
| Feature metadata | `kv-derived-refresh-receipt` | Lets live deployments advertise derived KV refresh receipt support |

## Operator Flow

```bash
contextdb repair kv-cache \
  --key context:prod:support:recent-nodes \
  --derive recent-nodes \
  --derive-namespace support \
  --derive-label SessionContext \
  --execute \
  --receipt-out context-prod-support-recent-nodes.receipt.json \
  --report
```

`--receipt-out` requires both `--execute` and `--derive recent-nodes`. The receipt stores the executed refresh report, value hash, and a doctor confirmation command.

## Compatibility

This release is intentionally non-breaking. Existing dry-run reports, literal/file KV refreshes, derived values, TTL behavior, overwrite behavior, and doctor checks are unchanged unless `--receipt-out` is supplied.
