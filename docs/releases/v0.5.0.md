# v0.5.0 Release Recap

v0.5.0 is a non-breaking release that makes feedback auditable. Validate, refute, useful, and stale operations still update node/source state as before, and now they also append durable feedback events.

## What Changed

| Area | New in v0.5.0 | Why it matters |
|:-----|:--------------|:---------------|
| Feedback event log | Durable `feedback` events for validate/refute/useful/stale | Creates an audit trail for operator and agent feedback |
| Go SDK reader | `NamespaceHandle.FeedbackEvents(ctx, after)` | Lets applications inspect feedback history without reaching into internal stores |
| REST audit endpoint | `GET /v1/namespaces/{ns}/feedback/events` | Makes feedback history available to operational tools |
| GraphQL audit query | `feedbackEvents(namespace:, after:)` | Gives debugger UIs a graph-friendly audit surface |
| Version metadata | `feedback-event-log` feature tag | Lets live deployments advertise support through introspection |

## Non-Breaking Behavior

Feedback methods keep their existing return values and state updates. Event records are additive and use the existing event-log store for embedded, standard, and remote modes.
