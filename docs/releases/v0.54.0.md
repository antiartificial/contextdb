# v0.54.0 Release Recap

v0.54.0 is a non-breaking durability release that adds opt-in KV hot-key sampling to `contextdb doctor`.

## What Changed

| Area | New in v0.54.0 | Why it matters |
|:-----|:---------------|:---------------|
| Doctor KV check | `contextdb doctor --kv-key KEY` | Lets operators verify expected hot cache keys |
| Repeated keys | Multiple `--kv-key` flags | Supports focused sampling for critical namespace cache entries |
| Refresh candidates | `kv_consistency` report | Missing keys are reported without mutating the cache |
| Feature metadata | `doctor-kv-consistency` | Lets live deployments advertise KV hot-key sampling support |

## Compatibility

This release adds a read-only doctor check. Existing doctor checks, REST APIs, storage behavior, and cache contents are unchanged.

## Operator Flow

```bash
CONTEXTDB_DATA_DIR=/var/lib/contextdb \
  contextdb doctor --kv-key context:my-app:active --kv-key context:my-app:summary
```

The check reports sampled keys, present keys, missing keys, and refresh candidates. It only calls `KVStore.Get` for configured keys.
