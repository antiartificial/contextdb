# v0.38.0 Release Recap

v0.38.0 is a non-breaking review workflow release that adds grouped escalation digests.

## What Changed

| Area | New in v0.38.0 | Why it matters |
|:-----|:---------------|:---------------|
| Go SDK digest | `ReviewEscalationDigest` | Summarizes escalated queue items without returning every task |
| REST digest | `GET /v1/namespaces/{ns}/review/escalations` | Gives dashboards grouped escalation counts using the same filters as the review queue |
| GraphQL digest | `reviewEscalationDigest(...)` | Gives debugger UIs grouped escalation data beside item-level queue fields |
| Grouping | Owner, source, item type, and escalation level | Helps operators route overdue work by responsibility and severity |
| Feature metadata | `review-escalation-digest` | Lets live deployments advertise grouped escalation summaries |

## Compatibility

This release only adds a new Go SDK helper, REST endpoint, GraphQL query, and feature metadata. Existing review queue callers and escalation metadata remain unchanged.

## Operator Flow

```bash
curl "http://localhost:7701/v1/namespaces/my-app/review/escalations?escalation_after_hours=72"
```

Digest groups include counts, max priority, max age, and review IDs:

```json
{
  "owner": "alice",
  "source_id": "docs-crawler",
  "type": "low_confidence",
  "escalation_level": "review_overdue",
  "count": 2
}
```
