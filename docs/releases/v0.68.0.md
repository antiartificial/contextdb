# v0.68.0 Release Recap

v0.68.0 is a non-breaking release-quality slice that adds reviewable deletion-plan output for versioned ranking eval baselines.

## What's New

| Area | New in v0.68.0 | Why it matters |
|:-----|:---------------|:---------------|
| Delete-plan output | `contextdb eval ranking --baseline-retention-dir DIR --emit-delete-script` | Converts pruneable ranking baseline artifacts into shell commands operators can inspect before cleanup |
| Artifact focus | Existing pruneable JSON and Markdown baselines only | Avoids commands for retained baselines, current baselines, missing files, or unrelated files |
| Report metadata | `delete_commands` in retention JSON | Lets automation review the same planned commands without parsing shell output |
| Feature metadata | `ranking-eval-baseline-delete-script` | Lets live deployments advertise reviewable ranking baseline cleanup support |

## Compatibility

This release only adds an optional output mode and additive JSON metadata. Existing ranking eval snapshots, Markdown recaps, baseline writes, baseline comparisons, retention reports, and ranking behavior are unchanged.

## Operator Flow

```bash
contextdb eval ranking \
  --baseline-retention-dir .contextdb/ranking-baselines \
  --baseline-retention-keep 5 \
  --emit-delete-script
```

Review the generated script before running it. contextdb does not delete baseline artifacts itself.
