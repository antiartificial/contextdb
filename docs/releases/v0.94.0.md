# v0.94.0 Release Recap

v0.94.0 is a non-breaking reliability release that adds a shared schema fixture for retry fatigue preset payloads.

## What's New

| Area | New in v0.94.0 | Why it matters |
|:-----|:---------------|:---------------|
| Preset schema fixture | `testdata/fixtures/retry-fatigue-presets.schema.json` | Documents the dashboard-facing retry fatigue preset JSON shape in a reusable artifact |
| SDK parity test | `TestReviewHandoffRetryFatiguePresetSchemaFixtureMatchesSDK` | Keeps `ReviewHandoffRetryFatiguePresets()` aligned with required fields, string values, and stable order |
| REST parity test | `TestRESTServer_RetryFatiguePresetSchemaFixture` | Confirms the public retry fatigue endpoint returns the same preset contract as the SDK |
| Feature metadata | `review-handoff-retry-fatigue-preset-schema-fixture` | Lets live deployments advertise preset schema fixture coverage through introspection |

## Operator Flow

Inspect retry fatigue presets from REST:

```bash
curl "http://localhost:7701/v1/namespaces/my-app/review/handoff-webhooks/retry-fatigue"
```

The `presets` array is now checked against the shared fixture used by SDK and REST tests.

## Compatibility

This release is intentionally non-breaking. Runtime behavior is unchanged; the release adds contract coverage and docs for the existing retry fatigue preset metadata.
