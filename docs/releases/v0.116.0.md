# v0.116.0

v0.116.0 is a non-breaking integration release that hardens Hermes review workflows around centralized ContextDB claim mutation.

## What's New

| Area | Surface | Why it matters |
|:-----|:--------|:---------------|
| Hermes tools | `hermes-memory-contextdb` | Adds review queue inspection and review-note annotation tools for agent memory workflows |
| Centralized mutation | ContextDB review worker | Keeps claim validation, refutation, and pruning in the durable worker instead of per application |
| Agent safety | Review notes and queue visibility | Lets Hermes participate in review workflows without granting direct mutation authority |

## Verification

- `go test ./...`
- `npm run docs:build`
- Hermes review integration coverage

## Compatibility

This release is additive. Hermes review tools add inspection and annotation paths while existing memory writes and ContextDB worker policies remain unchanged.
