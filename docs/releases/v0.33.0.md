# v0.33.0 Release Recap

v0.33.0 is a non-breaking operations release that adds verification for lifecycle manifest indexes.

## What's New

| Area | New in v0.33.0 | Why it matters |
|:-----|:---------------|:---------------|
| Index verification | `contextdb snapshot lifecycle index verify --in PATH --report` | Re-checks a saved backup catalog against files on disk |
| Artifact checks | Verifies indexed paths, byte sizes, and SHA-256 hashes | Catches missing, truncated, or changed artifacts after indexing |
| JSON report | Verification reports per-artifact results and validation errors | Gives operators an audit artifact for backup reviews |
| Failure behavior | Drift exits non-zero | Makes the command useful in automation |
| Feature metadata | `lifecycle-index-verify` | Lets live deployments advertise index verification through introspection |

## Compatibility

This release only adds a new lifecycle index verification command and feature metadata. Existing lifecycle indexes, lifecycle reports, deletion plans, backup, restore, doctor, Norn, REST, GraphQL, and SDK behavior is unchanged.
