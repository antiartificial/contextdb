# v0.21.0 Release Recap

v0.21.0 is a non-breaking operations release that turns the snapshot and doctor pieces into a scheduled backup workflow. The new backup runbook documents export, restore preview, backup marker verification, launchd/systemd scheduling, and Norn drift pairing.

## What's New

| Area | New in v0.21.0 | Why it matters |
|:-----|:---------------|:---------------|
| Backup runbook | [Deployment: Backup Runbook](../deployment/backup-runbook) | Gives operators one concrete daily flow instead of scattered commands |
| Restore preview gate | Dry-run import with `--report` before retention or promotion | Makes backup contents and node diffs visible before trust is placed in the file |
| Scheduler examples | launchd and systemd snippets | Makes the live Mini/Norn path and Linux deployments easier to automate |
| Norn pairing | Drift check next to doctor backup freshness | Separates service-discovery drift from backup-readiness evidence |
| Feature metadata | `backup-runbook` | Lets deployments advertise that the documented operational flow is part of the release surface |

## Compatibility

This release only adds documentation and feature metadata. Runtime behavior, APIs, CLI flags, and storage formats are unchanged.
