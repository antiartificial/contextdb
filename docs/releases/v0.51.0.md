# v0.51.0 Release Recap

v0.51.0 is a non-breaking review operations release that adds read-only retry fatigue summaries for failed review handoff webhooks.

## What Changed

| Area | New in v0.51.0 | Why it matters |
|:-----|:---------------|:---------------|
| Go SDK summary | `ReviewHandoffRetryFatigue` | Groups unresolved retry pressure by target endpoint |
| REST summary | `GET /v1/namespaces/{ns}/review/handoff-webhooks/retry-fatigue` | Gives operators a dashboard-friendly endpoint view |
| GraphQL summary | `reviewHandoffRetryFatigue(...)` | Lets inspector UIs surface noisy or stuck handoff targets |
| Feature metadata | `review-handoff-retry-fatigue` | Lets live deployments advertise endpoint-level retry summaries |

## Compatibility

This release adds read-only summary APIs. It does not schedule retries, send webhooks, change retry backoff calculations, or alter delivery receipt storage.

## Operator Flow

```bash
curl "http://localhost:7701/v1/namespaces/my-app/review/handoff-webhooks/retry-fatigue"
```

Each summary includes the target URL, unresolved candidate count, total attempts, ready and waiting recommendation counts, status-family counts, and the latest failure detail.
