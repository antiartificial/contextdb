# v0.17.0 Release Recap

v0.17.0 is a non-breaking durability release that productizes snapshot backup and restore. The internal NDJSON snapshot machinery is now available through public Go client helpers and `contextdb snapshot` commands, including dry-run import validation.

## What's New

| Area | New in v0.17.0 | Why it matters |
|:-----|:---------------|:---------------|
| Go client snapshots | `ExportSnapshot`, `ExportSnapshotFromSeeds`, `ImportSnapshot`, and `ValidateSnapshot` | Gives applications a supported backup/restore API without importing internal packages |
| CLI export | `contextdb snapshot export --namespace my-app --out backup.ndjson` | Lets operators create namespace backups from the same environment settings used by the server |
| Filtered export | `--seeds` and `--max-depth` | Supports focused subgraph backups for a claim, incident, or migration preview |
| CLI import dry run | `contextdb snapshot import --dry-run` | Validates NDJSON shape and vector reindexing without writing to the target DB |
| Full export repair | Namespace export now uses `ValidAt` plus node history and outgoing edges | Makes unseeded namespace export useful for the built-in stores |
| Feature metadata | `snapshot-backup-restore` feature tag | Lets live deployments advertise snapshot backup/restore support |

## Compatibility

All APIs and commands are additive. Snapshot imports still override record namespaces with the requested target namespace, so operators can restore into a preview namespace before importing into the production namespace.
