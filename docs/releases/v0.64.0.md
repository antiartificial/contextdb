# v0.64.0 Release Recap

v0.64.0 is a non-breaking release-operations slice that adds read-only retention reports for versioned ranking baselines.

## What Changed

| Area | New in v0.64.0 | Why it matters |
|:-----|:---------------|:---------------|
| Baseline retention report | `contextdb eval ranking --baseline-retention-dir DIR` | Lists versioned ranking baseline artifacts without running a new eval |
| Retention policy | `--baseline-retention-keep N` | Marks the newest N baseline versions as retained and older versions as pruneable |
| Current marker | Newest baseline version is marked `current` | Makes release-review baselines easier to inspect before pruning |
| Feature metadata | `ranking-eval-baseline-retention` | Lets live deployments advertise support for ranking baseline retention inspection |

## Compatibility

This release only adds optional ranking eval flags and feature metadata. Existing snapshots, Markdown recaps, baseline writes, baseline comparisons, and diff output behavior are unchanged.

## Operator Flow

```bash
contextdb eval ranking \
  --baseline-retention-dir .contextdb/ranking-baselines \
  --baseline-retention-keep 5
```

The report is read-only: it identifies retained and pruneable baseline versions, but does not delete files.
