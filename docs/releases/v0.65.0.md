# v0.65.0 Release Recap

v0.65.0 is a non-breaking documentation release that adds cookbook guidance for retry fatigue owner and escalation-lane filters.

## What Changed

| Area | New in v0.65.0 | Why it matters |
|:-----|:---------------|:---------------|
| Cookbook page | `docs/deployment/retry-fatigue-cookbook.md` | Gives operators copyable REST, Markdown, and GraphQL examples for scoped fatigue handoffs |
| Owner lane examples | `owner=alice` | Shows how to focus unresolved retry pressure on one workload owner |
| Escalation lane examples | `escalation_level=review_overdue` | Shows how to route retry fatigue by escalation class |
| Feature metadata | `review-handoff-retry-fatigue-cookbook` | Lets live docs advertise the handoff-lane cookbook surface |

## Compatibility

This release adds docs and feature metadata only. Existing retry fatigue JSON, Markdown export, REST filters, GraphQL filters, Go SDK helpers, retry recommendations, retry execution, and delivery receipts are unchanged.

## Operator Flow

```bash
curl "http://localhost:7701/v1/namespaces/my-app/review/handoff-webhooks/retry-fatigue?owner=alice&escalation_level=review_overdue&format=markdown"
```

Use the cookbook when a broad endpoint fatigue report needs to become a narrower owner or escalation-lane handoff.
