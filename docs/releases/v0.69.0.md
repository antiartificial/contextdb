# v0.69.0 Release Recap

v0.69.0 is a non-breaking review operations release that adds stable preset names for repeated retry fatigue lanes.

## What's New

| Area | New in v0.69.0 | Why it matters |
|:-----|:---------------|:---------------|
| REST presets | `retry-fatigue?preset=review-overdue` | Gives dashboards and scripts a stable lane name instead of repeating raw filter values |
| GraphQL presets | `reviewHandoffRetryFatigue(preset: "review-overdue")` | Keeps dashboard queries aligned with REST handoff lanes |
| Go SDK presets | `ReviewHandoffRetryFatigueRequest{Preset: "review-overdue"}` | Lets SDK callers reuse the same named lanes |
| Preset metadata | `presets` in REST JSON responses | Lets clients discover built-in lane names and descriptions |
| Feature metadata | `review-handoff-retry-fatigue-presets` | Lets live deployments advertise preset-aware retry fatigue support |

## Compatibility

This release only adds optional filters and additive response metadata. Existing `owner`, `escalation_level`, GraphQL `owner` and `escalationLevel`, Markdown export, retry recommendations, retry execution, and delivery receipts are unchanged.

## Operator Flow

```bash
curl "http://localhost:7701/v1/namespaces/my-app/review/handoff-webhooks/retry-fatigue?preset=review-overdue"
```

Built-in presets are `review-overdue`, `source-trust-anomaly`, and `unassigned-review-overdue`. Explicit owner or escalation filters still work and can override preset-expanded values.
