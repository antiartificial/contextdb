# v0.70.0 Release Recap

v0.70.0 is a non-breaking operations documentation release that adds a safety guard for published backup catalog repair.

## What's New

| Area | New in v0.70.0 | Why it matters |
|:-----|:---------------|:---------------|
| Guard page | `docs/deployment/published-backup-repair-guard.md` | Gives operators a checklist before replacing published backup catalog metadata |
| Dry-run review | Recommended publish command remains dry-run first | Keeps drift repair inspectable before writes |
| Execute checks | Verification, off-host copy, retention review, environment URL, token scope, and saved report checks | Reduces accidental replacement of the published catalog |
| Post-repair checks | Drift and freshness commands after execution | Confirms dashboards point at the intended backup catalog |
| Feature metadata | `published-backup-repair-guard` | Lets live docs advertise the repair guard surface |

## Compatibility

This release adds docs and feature metadata only. Published backup drift reports, recommended dry-run commands, publish execution, doctor checks, and freshness checks are unchanged.

## Operator Flow

```bash
contextdb doctor \
  --published-backup-index "$CONTEXTDB_BACKUP_DIR/contextdb-backups.index.json" \
  --published-backup-url "$CONTEXTDB_LIFECYCLE_INDEX_PUBLISHED_URL" \
  --report
```

When doctor reports only `published_backup_drift`, run the recommended publish command as a dry run, complete the guard checklist, then add `--execute --token "$NORN_TOKEN"` only after review.
