# v0.90.0 Release Recap

v0.90.0 is a non-breaking reliability release for ranking manifest verification bundles.

## What's New

| Area | New in v0.90.0 | Why it matters |
|:-----|:---------------|:---------------|
| Bundle index verifier | `contextdb eval ranking baseline manifest bundle verify --index PATH --report` | Re-checks archived verification bundles without regenerating them |
| Artifact integrity | Byte-size and SHA-256 checks for indexed JSON, Markdown, and annotation files | Catches edited, missing, or mismatched bundle artifacts in CI or release review |
| Report consistency | Confirms the index `ok` and `status` match the JSON verification report | Prevents stale pointer files from claiming a different outcome than the report |
| Feature metadata | `ranking-baseline-bundle-index-verify` | Lets live deployments advertise bundle index verification support |

## Operator Flow

Create a verification bundle:

```bash
contextdb eval ranking baseline manifest verify \
  --manifest ranking-baseline-manifest.json \
  --bundle-dir ranking-baseline-verification
```

Verify the archived bundle index later:

```bash
contextdb eval ranking baseline manifest bundle verify \
  --index ranking-baseline-verification/ranking-baseline-manifest-verification-index.json \
  --report
```

## Compatibility

This release is intentionally non-breaking. Existing ranking manifest verification, bundle output, admin dashboard, debugger, REST, GraphQL, and SDK behavior are unchanged.
