# v0.15.0 Release Recap

v0.15.0 is a non-breaking review workflow release that adds focused review queue filters. Operators and debugger UIs can now narrow derived review tasks by item type, source, workflow status, and owner without changing how queue items are generated.

## What's New

| Area | New in v0.15.0 | Why it matters |
|:-----|:---------------|:---------------|
| Go SDK filters | `ReviewQueueRequest.Types`, `SourceID`, `Status`, and `Owner` | Lets applications build focused review views without post-filtering every queue response |
| REST filters | `type`, `source_id`, `status`, and `owner` query parameters | Gives operators simple URLs for source anomaly, assigned, and open review queues |
| GraphQL filters | `reviewQueue(types:, sourceId:, status:, owner:)` | Gives debugger UIs first-class controls for reviewer workflows |
| Open-state matching | Items without a durable decision match `status=open` | Keeps derived queue behavior stateless while making default queue state queryable |
| Feature metadata | `review-queue-filters` feature tag | Lets live deployments advertise focused review queue filtering support |

## Compatibility

All request fields and query arguments are additive. Existing review queue callers continue to receive the same ranked items unless they opt into one or more filters. Resolved tasks remain hidden, and snoozed tasks remain hidden until their re-check time.
