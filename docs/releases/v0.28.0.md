# v0.28.0 Release Recap

v0.28.0 is a non-breaking operations release that bundles the backup lifecycle into one documented operator workflow.

## What's New

| Area | New in v0.28.0 | Why it matters |
|:-----|:---------------|:---------------|
| Lifecycle script | Backup runbook script for export, verify, rehearse, doctor, optional promote, receipt verify, and summary output | Gives operators one copyable chain for the full backup lifecycle |
| Guarded promotion | Promotion only runs when `PROMOTE=1` is set | Keeps the default flow non-mutating beyond backup creation |
| Lifecycle summary | Final JSON file records backup, manifest, rehearsal, promotion, receipt check, and promotion status paths | Makes automation output easy to archive |
| Scheduler fit | Script remains compatible with launchd and systemd examples | Lets the same runbook serve Mini/Norn and Linux deployments |
| Feature metadata | `backup-lifecycle-bundle` | Lets deployments advertise the lifecycle workflow in docs and introspection |

## Compatibility

This release adds documentation and feature metadata only. Existing snapshot commands, receipt verification, manifests, markers, doctor checks, and restore flows keep their behavior unchanged.
