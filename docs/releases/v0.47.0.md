# v0.47.0 Release Recap

v0.47.0 is a non-breaking backup operations release that adds a published catalog drift watch for lifecycle index metadata.

## What Changed

| Area | New in v0.47.0 | Why it matters |
|:-----|:---------------|:---------------|
| CLI drift watch | `contextdb snapshot lifecycle index publish drift` | Compares a local lifecycle index publish payload with the published ops payload |
| Published payload fetch | `--published-url` plus optional bearer token | Lets operators check the live metadata endpoint without uploading backup contents |
| Drift report | `--report` JSON output | Shows count, retention decision, artifact count, byte, and hash coverage differences |
| Feature metadata | `lifecycle-index-publish-drift` | Lets live deployments advertise backup publish drift support |

## Compatibility

This release adds a read-only CLI check. It does not change snapshot export/import, lifecycle index generation, publish payload shape, or existing publish execution.

## Operator Flow

```bash
contextdb snapshot lifecycle index publish drift \
  --in "$CONTEXTDB_BACKUP_DIR/contextdb-backups.index.json" \
  --published-url "$CONTEXTDB_LIFECYCLE_INDEX_PUBLISHED_URL" \
  --token "$NORN_TOKEN" \
  --report
```

The command exits non-zero when published metadata differs from the local lifecycle index publish payload.
