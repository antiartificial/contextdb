# v0.37.0 Release Recap

v0.37.0 is a non-breaking review workflow release that adds escalation metadata to review queue items.

## What Changed

| Area | New in v0.37.0 | Why it matters |
|:-----|:---------------|:---------------|
| Review escalation | `ReviewQueueRequest.EscalationAfter` | Flags assigned or due snoozed review tasks that have waited beyond an operator-defined age threshold |
| Source anomaly escalation | `SourceAnomalyEscalationPriority` | Lets high-priority source-trust anomalies escalate when they age past the same threshold |
| REST controls | `escalation_after_hours` and `source_anomaly_escalation_priority` | Gives operations dashboards URL-level escalation controls |
| GraphQL controls | `escalationAfterHours` and `sourceAnomalyEscalationPriority` | Gives debugger UIs first-class review escalation fields |
| Feature metadata | `review-escalation-rules` | Lets live deployments advertise escalation support through introspection |

## Compatibility

This release only adds optional request fields and optional response metadata. Existing review queue callers keep receiving the same items unless they opt into escalation thresholds.

## Operator Flow

```bash
curl "http://localhost:7701/v1/namespaces/my-app/review/queue?status=assigned&escalation_after_hours=72"
```

Escalated items include:

```json
{
  "escalated": true,
  "escalation_level": "review_overdue",
  "escalation_reason": "assigned review has waited 72.0 hours",
  "escalation_age_hours": 72
}
```
