# v0.16.0 Release Recap

v0.16.0 is a non-breaking operations release that adds a live Norn manifest drift check. Operators can now compare the locally generated contextdb service entry with the live Norn manifest and get field-level differences before a route, port, version, or tag mismatch becomes harder to spot.

## What's New

| Area | New in v0.16.0 | Why it matters |
|:-----|:---------------|:---------------|
| Drift command | `contextdb norn drift --manifest-url "$NORN_MANIFEST_URL" --endpoint "$CONTEXTDB_URL"` | Compares expected service metadata with the live Norn manifest |
| Manifest decoding | Supports a single service object, an array of services, or an object with `services` | Fits common Norn manifest response shapes without extra adapters |
| Field-level report | Reports diffs for `endpoint`, URLs, version, ports, and tags | Makes deployment drift actionable rather than a vague health failure |
| CI-friendly exit | Exits zero on match and non-zero on drift | Lets release or Mini operations scripts fail clearly when Norn is stale |
| Feature metadata | `norn-live-drift-check` feature tag | Lets live deployments advertise drift check support |

## Compatibility

The command is additive and does not change server behavior, manifest generation, or validation. Existing `contextdb norn manifest` and `contextdb norn validate` workflows continue to work unchanged.
