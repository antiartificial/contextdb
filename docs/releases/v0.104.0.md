# v0.104.0 Release Recap

v0.104.0 is a non-breaking product and inspectability release that turns the belief debugger into a richer epistemics workbench.

## What's New

| Area | New in v0.104.0 | Why it matters |
|:-----|:----------------|:---------------|
| Epistemics envelope | `/admin/api/belief` now includes an additive `epistemics` object | Keeps the raw audit response compatible while giving dashboards stable visualization fields |
| Source trust timeline | The debugger renders source credibility events over time | Operators can see how validation/refutation history changed source trust |
| Confidence history | The debugger visualizes node confidence versions | Makes claim-level belief drift visible without reading raw JSON |
| Contradiction paths | Contradicting nodes are summarized as inspectable path rows | Speeds up conflict triage and can hand off directly to explain-rank compare |
| Graph/source context | Neighbor nodes are listed with relation, source, confidence, and edge weight | Lets operators walk evidence, support, provenance, and contradiction context from one screen |
| Feature metadata | `admin-epistemics-visualization` | Lets live deployments advertise the richer debugger surface through version introspection |

## Operator Flow

Open the observe admin UI:

```bash
open http://localhost:7702/admin/
```

Search for a node in the Belief Debugger, inspect it, then use the rendered timeline, contradiction paths, and graph context to follow evidence without copying UUIDs between tools.

The raw audit JSON remains available in a collapsible panel for scripts and low-level debugging.

## Compatibility

This release is intentionally non-breaking. Existing `/admin/api/belief` callers still receive the previous raw audit fields; the new `epistemics` object is additive.

## Verification

```bash
npm run admin:build
```
