# v0.31.0 Release Recap

v0.31.0 is a non-breaking operations release that adds reviewable deletion-plan output for backup lifecycle retention.

## What's New

| Area | New in v0.31.0 | Why it matters |
|:-----|:---------------|:---------------|
| Delete-plan output | `contextdb snapshot lifecycle retention --emit-delete-script` | Converts the dry-run retention report into shell commands operators can inspect before cleanup |
| Non-destructive safety | contextdb still never deletes files | Keeps cleanup approval outside the tool by default |
| Artifact focus | Only existing artifacts from `pruneable` bundles are included | Avoids commands for kept bundles or already-missing files |
| Stable scripting | Commands are emitted as `rm -- 'path'` with shell quoting | Makes cleanup plans easier to paste into change reviews |
| Feature metadata | `lifecycle-delete-plan` | Lets live deployments advertise delete-plan support through introspection |

## Compatibility

This release only adds a retention output mode and feature metadata. Existing lifecycle reports, lifecycle verification, backup, restore, doctor, Norn, REST, GraphQL, and SDK behavior is unchanged.
