# v0.4.1 Release Recap

v0.4.1 is a non-breaking patch release that expands `contextdb doctor` from metadata-only checks into an opt-in live write/read probe.

## What Changed

| Area | New in v0.4.1 | Why it matters |
|:-----|:--------------|:---------------|
| Sample write probe | `contextdb doctor --sample-write` | Verifies the REST write path, vector retrieval path, and index visibility against a live deployment |
| Sample namespace control | `--sample-namespace` | Lets operators choose where probe nodes are written; defaults to `_doctor` |
| Doctor report metadata | `sample_node_id` in the JSON report | Makes the probe result auditable and easy to inspect after a run |
| Doctor tests | Success and retrieve-miss coverage | Locks down both the healthy path and the failure signal for index drift |

## Non-Breaking Behavior

`contextdb doctor` remains non-mutating by default. The write/read probe only runs when `--sample-write` is passed. Probe writes opt into content deduplication and use the `DoctorProbe` label.
