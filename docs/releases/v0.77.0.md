# v0.77.0 Release Recap

v0.77.0 is a non-breaking review operations release that adds copyable filter examples to retry fatigue preset metadata.

## What's New

| Area | New in v0.77.0 | Why it matters |
|:-----|:---------------|:---------------|
| Preset metadata | `example_rest_query` | Lets dashboards render a copyable REST query string for each built-in retry fatigue preset |
| GraphQL handoff | `example_graphql` | Gives dashboard builders the matching GraphQL argument snippet |
| Cookbook table | REST and GraphQL example columns | Keeps human docs aligned with the metadata surfaced by REST |
| Feature metadata | `review-handoff-retry-fatigue-preset-examples` | Lets live deployments advertise preset example support |

## Operator Flow

Fetch retry fatigue summaries and preset metadata:

```bash
curl "http://localhost:7701/v1/namespaces/my-app/review/handoff-webhooks/retry-fatigue"
```

Each item in `presets` now includes examples like:

```json
{
  "name": "review-overdue",
  "escalation_level": "review_overdue",
  "example_rest_query": "preset=review-overdue",
  "example_graphql": "preset: \"review-overdue\""
}
```

## Compatibility

This release is intentionally non-breaking. Existing preset names, owner and escalation filters, REST query parameters, GraphQL arguments, and retry fatigue summaries are unchanged.
