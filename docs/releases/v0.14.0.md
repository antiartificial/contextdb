# v0.14.0 Release Recap

v0.14.0 is a non-breaking operations release that adds a local Norn registration helper. Operators can now generate and validate the contextdb service entry using the same default ports and public endpoint assumptions documented for the Mini/Norn deployment.

## What Changed

| Area | New in v0.14.0 | Why it matters |
|:-----|:---------------|:---------------|
| Manifest generation | `contextdb norn manifest` | Produces a contextdb service entry with endpoint, health, GraphQL, features, ports, tags, and version |
| Manifest validation | `contextdb norn validate --file PATH` | Catches wrong app ids, missing service names, relative endpoints, and missing REST ports before registration |
| Environment support | `CONTEXTDB_PUBLIC_URL`, `CONTEXTDB_GRPC_ADDR`, `CONTEXTDB_REST_ADDR`, `CONTEXTDB_OBS_ADDR` | Reduces drift between server runtime config, Norn entries, and docs |
| Feature metadata | `norn-registration-helper` feature tag | Lets live deployments advertise that this operational helper is available |

## Non-Breaking Behavior

The helper is additive. It does not change server startup, existing Norn registrations, or doctor behavior.
